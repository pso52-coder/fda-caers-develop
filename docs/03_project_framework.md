# 03. 프로젝트 골자 정리 — NASA 레퍼런스 방식 적용

이 문서는 NASA C-MAPSS 예지보전 프로젝트에서 사용한 기본 골자를 FDA CAERS 프로젝트에 맞게 변환한 기록입니다.

## 1. 문제 정의

| 구분 | NASA C-MAPSS | FDA CAERS 프로젝트 |
| --- | --- | --- |
| 현장 문제 | 엔진 고장 시점을 몰라 정비 비용과 안전 리스크 발생 | 식품 부작용 신고가 누적되지만 조사 우선순위 판단이 어려움 |
| 사용자 | 항공사 정비 담당자 | 식품 안전 모니터링/품질 리스크 담당자 |
| 최종 행동 | 정비 시점 결정 | 우선 조사 브랜드/식품군 선별 |
| 핵심 출력 | RUL, 조기 경보 | watch_score, 급증 연도, 고위험율 |

## 2. ML/분석 문제로 번역

| 관점 | 정의 |
| --- | --- |
| Target 또는 판단 기준 | 브랜드별 신고 급증 여부, 고위험 결과 비율, 통합 watch_score |
| 문제 유형 | EDA + 비지도 규칙 기반 이상 신호 탐지 |
| 평가 방식 | 실제 리콜 사례와 신호 발생 연도 정렬 여부 확인 |
| 운영 기준 | 위험 확정이 아니라 조사 우선순위 제공 |

## 3. 데이터 이해

| 항목 | 내용 |
| --- | --- |
| 데이터 | FDA CAERS 식품 부작용 신고 데이터 |
| 기간 | 2004년~2017년 Q2 |
| 단위 | 신고 건 / 브랜드 / 식품군 / 연도 |
| 주요 피처 | 브랜드, 산업군, 증상, 결과, 나이, 성별, 신고일 |
| 주요 한계 | 자발신고 편향, 판매량 분모 부재, 미디어 효과 |

## 4. 분석 흐름

```text
원본 신고 데이터
→ 전처리 기준 보정
→ serious/severe 심각도 정의
→ 식품군·연령대 파생
→ 식품군별 EDA
→ 브랜드별 연도 신고 건수 집계
→ 급증 신호 탐지
→ 고위험율과 결합한 watch_score 산출
→ 실제 리콜 사례 검증
→ 대시보드화
```

## 5. 포트폴리오에서 강조할 지점

1. **문제 정의가 명확함**  
   “식품 부작용 데이터를 분석했다”가 아니라 “조사 우선순위가 높은 브랜드를 자동 선별했다”로 정의했습니다.

2. **데이터 기준을 보정함**  
   serious 정의, 식품군 분류, 나이 변환 기준을 문서화했습니다.

3. **EDA에서 자동 탐지로 확장함**  
   팀이 수작업으로 찾은 위험 브랜드를 규칙 기반 탐지기로 재현했습니다.

4. **실제 사건과 검증함**  
   OxyElite Pro, Hydroxycut 등 실제 리콜 사건과 급증 연도를 대조했습니다.

5. **운영 한계를 명시함**  
   자발신고 데이터의 특성상 위험률이 아니라 조사 우선순위로 해석해야 함을 명확히 했습니다.
